## 2. 348A-R1：Evidence Policy Refinement

### 问题

第一版把所有只有 workbook lineage 的行都压成 `missing_evidence`。

但这些行并不是完全没有来源，而是缺少强页码 / 截图 / 坐标证据。

### 修改方向

引入 evidence level：

```text
STRONG_EVIDENCE
WEAK_EVIDENCE
MISSING_EVIDENCE
NOT_APPLICABLE
```

### 结果

```text
missing_evidence_count: 82 -> 0
weak_evidence_count = 82
```

### 结论

R1 解决了“弱证据”和“真正缺证据”混在一起的问题。

## 4. 348A-R3：Unit Checker False Positive Refinement

### 问题

`净资产收益率(%)` 被误判为 `monetary_unit_mismatch`。

原因是 label 中包含 `资产`，unit checker 将其当作货币金额类指标。

### 修复方向

将 ROE / 净资产收益率 / 收益率 / 毛利率 / 净利率等识别为 percentage/rate metric。

### 结果

```text
unit_issue_count: 1 -> 0
fail_count: 1 -> 0
review_count = 82
```

### 结论

R3 修掉第一样本的明确 unit false positive，同时保留 `营业收入(%)`、`资产总计(%)` 等真正错误场景的检查能力。

## 5. 348A-R4：Clean Data Candidate Policy

### 问题

R3 后所有行仍然是 `WEAK_EVIDENCE`，如果完全按正式交付标准处理，clean data 仍为空。

### 核心原则

```text
internal clean-data candidate ≠ client_ready
internal clean-data candidate ≠ production_ready
internal clean-data candidate ≠ formal client delivery
```

### 规则

```text
STRICT_FINANCIAL_TABLE_ROW + WEAK_EVIDENCE + 无 unit/period/valuation issue
=> INTERNAL_CLEAN_CANDIDATE

MARKET_REFERENCE_ROW + WEAK_EVIDENCE + 无 unit issue
=> INTERNAL_REFERENCE_CANDIDATE

NARRATIVE_ASSERTION
=> NARRATIVE_REVIEW，不进 clean_data

MISSING_EVIDENCE
=> REVIEW / FAIL，不进 clean_data
```

### 结果

```text
clean_data_row_count = 75
review_queue_row_count = 7
internal_clean_candidate_count = 65
internal_reference_candidate_count = 10
narrative_review_count = 5
review_required_count = 2
client_ready = false
production_ready = false
formal_client_export_allowed = false
```

### 结论

R4 让第一真实样本产生内部可用 clean-data candidate，但没有打开正式交付 gate。

## 9. 348S-R1-QA：Intake Generalization Review

### QA 重点

检查 R1 是否是真改善，不是靠漏读数据刷指标。

重点问题：

```text
row_count_total 119 -> 112 的 7 行差异是否合理
UNKNOWN_ROW = 0 是否可信
剩余 5 个 period issue 是否是真问题
剩余 2 个 unit issue 是否是真问题
clean_data 87 行是否干净
baseline 是否无回退
```

### QA 结论

`119 -> 112` 的 7 行差异合理，来自 7 个结构化 sheet 各去掉 1 行 header-like pollution row：

```text
盈利预测与估值: 7 -> 6
资产负债表: 31 -> 30
利润表: 27 -> 26
现金流量表: 8 -> 7
重要财务与估值指标: 8 -> 7
可比公司估值: 5 -> 4
盈利预测分业务: 6 -> 5
```

被去掉的都是 `指标` / `项目` / `公司` / `业务板块` 这类 header-like 行，不是有效业务数据。

剩余问题被确认更像误报：

```text
5 个 period issue：盈利预测分业务 的 embedded period header 未识别
2 个 unit issue：资产负债率(%) / 资产负债率(%,LF) 被误判为 monetary metric
```

### 结论

R1-QA 确认 intake generalization 是有效改善。

推荐下一步：

```text
348S-R2 Unit/Period Residual Refinement
```
