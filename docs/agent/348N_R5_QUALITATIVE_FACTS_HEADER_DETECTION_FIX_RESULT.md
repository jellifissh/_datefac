## Implementation summary

The R4 diagnosis identified the root cause as an intake-layer header-detection defect, not a clean-candidate-policy problem. R5 fixes it at the intake layer using the same named-header detection pattern already used for `data_dictionary` / `doc_metadata` / `figure_index` / `related_research` / `market_base_data` / `validation_checks`.

Three coordinated changes in `datefac_agent/intake/excel_intake.py`:

1. New named-header constant `QUALITATIVE_FACTS_HEADERS = {"事实ID", "页码", "类别", "指标/事件", "数值", "单位", "期间", "摘录/说明", "置信度"}`.
2. Added `qualitative_facts` to the `_find_special_header_row` `header_lookup`, so its real Chinese header row is matched via `_matches_named_header` (subset match) before the generic `_is_header_candidate` can misfire on the F001 data row.
3. Added `qualitative_facts` to `TESTSET_SUPPORTING_SHEETS` (facts-schema rows route to `TESTSET_SUPPORTING_ROW`, not a financial wide-table type) and added a `qualitative_facts` branch in `_extract_special_metric_name` that prefers `指标/事件` then `事实ID`.

This deliberately does NOT touch `clean_candidate_policy.py`: the rows leave `clean_data` through the existing policy because `TESTSET_SUPPORTING_ROW -> REVIEW_REQUIRED`, exactly as R4 recommended.

## Header detection fix details

Pre-R5, `_find_special_header_row` had no entry for `qualitative_facts`, so intake fell back to the generic `_find_header_row`. `_is_header_candidate` skipped the real row-1 header (`事实ID` is not in `HEADER_LABEL_HINTS` and the header has no period-style cells), then accepted the F001 data row as the header because `1995` and `公司成立于1995年…` both matched `PERIOD_LABEL_RE` (`period_hits >= 2`).

Post-R5, `_find_special_header_row` matches the real header on row 1 via the named-header subset check. Verified directly: `_find_special_header_row("qualitative_facts", sheet_rows)` now returns `(1, ['事实ID','页码','类别','主体','指标/事件','数值','单位','期间','摘录/说明','置信度'])`, and F001 is parsed as a data row, not a header.

This is sheet/header-family based, not filename based, and it does not loosen `_is_header_candidate` for ordinary data rows — the named-header path only fires when the specific header set is present.
