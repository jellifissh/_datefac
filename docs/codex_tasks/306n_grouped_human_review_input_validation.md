# 306N Grouped Human Review Input Validation

## Goal
- Validate grouped human review input generated by 306M.

## Constraints
- Do not rerun Marker.
- Do not rerun pdfplumber.
- Do not call API / LLM / OCR.
- Do not modify production.

## Inputs
- `output/eval_306m_grouped_human_review_input_design/306m_group_id_manifest.xlsx` (immutable group source)
- `output/eval_306m_grouped_human_review_input_design/306m_group_to_candidate_manifest.xlsx` (candidate mapping source)
- `output/eval_306m_grouped_human_review_input_design/306m_sample_grouped_review_input.xlsx` (sample input)
- optional real input: `input/human_review/306n_grouped_human_review_input.xlsx`

## Validation Rules
1. `group_id` exists in manifest.
2. `group_id` not duplicated (for reviewed rows).
3. Immutable group fields unchanged vs group manifest.
4. `decision` in:
   - approve_series
   - reject_series
   - needs_more_info
   - correct_series
5. `reviewer_id` required for reviewed rows.
6. `reviewed_at` required and parseable for reviewed rows.
7. `correct_series` requires at least one corrected year field (`corrected_2020...corrected_2030`) or `corrected_unit`.
8. `needs_more_info` requires `extra_info_request`.
9. `reject_series` should include `review_comment`.
10. Forbidden fields absent:
   - safe_to_apply
   - approve_for_real_apply
11. Group-to-candidate mapping row count remains 372.

## Outputs
- `output/eval_306n_grouped_human_review_input_validation/`
  - `306n_summary.json`
  - `306n_report.md`
  - `306n_valid_group_review_results.xlsx`
  - `306n_invalid_group_review_results.xlsx`
  - `306n_group_review_validation_audit.xlsx`
  - `306n_negative_validation_tests.json`
  - `306n_no_apply_proof.json`

## Assertions
- If real input missing: validate sample only and set `real_input_present=false`.
- check_delivery_state must remain PASS.
- production/official/formal/standardizer/release unchanged.
