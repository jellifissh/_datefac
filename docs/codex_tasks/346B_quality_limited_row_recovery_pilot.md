## Recovery logic

### Layer 1: deterministic value sanitizer

Apply deterministic cleaning to raw values without overwriting originals.

Handle at least:

```text
1,200.00 -> 1200.00
(15.4) -> -15.4
12.5% -> parseable percent marker
-- / — / N/A / 不适用 -> normalized missing marker
full-width spaces / non-breaking spaces / zero-width spaces -> normalized
```

Recommended output fields:

```text
raw_value
sanitized_value
value_parse_status
value_parse_error
value_numeric_type
```

### Layer 2: context state machine / inheritance

Build scoped context from row fields, JSON/MD context snippets, table body, captions, footnotes, neighbor rows, and table image metadata.

Context scopes:

```text
PDFContext
PageContext
TableContext
HeaderContext
RowContext
```

Attempt to infer or inherit:

```text
unit
currency
period
period_type
statement_type
source trace / table evidence
```

Examples:

- `营业收入（百万元）` implies unit `百万元` for that metric row.
- `每股收益（元）` implies unit `元`.
- `EV/EBITDA` normally behaves as a multiple/ratio; missing monetary unit may be a false-positive unit issue and should be marked as `UNIT_NOT_APPLICABLE_RATIO_MULTIPLE` instead of blindly filling a currency unit.
- `PE`, `PB`, `ROE`, `EBIT Margin`, and margin/rate rows should not inherit `百万元` from nearby revenue rows.

Do not inject table-level monetary units into ratio, margin, multiple, percentage, or per-share rows without rule support.

Recommended fields:

```text
inherited_unit
unit_repair_action
unit_repair_source
unit_repair_confidence
period_repair_action
period_repair_source
period_repair_confidence
context_notes
```

### Layer 3: text + image evidence assisted recovery

For rows with 346A2 table crop image binding or JSON/MD context binding, record evidence availability.

Do not inspect image pixels directly in 346B unless an existing deterministic image metadata/table_body path already contains extractable text context. 346B may use MinerU JSON/MD table_body and text context, but it must not invoke OCR or VLM.

Recommended fields:

```text
image_bound
image_evidence_type
chosen_image_path
json_context_path
md_context_path
context_available
neighbor_context_rows_count
evidence_assisted_recovery_action
evidence_confidence
```

### Layer 4: re-audit and re-rank

Each pilot row must be re-ranked into one of:

```text
RECOVERED_DEMO_CANDIDATE
STILL_QUALITY_LIMITED
NEEDS_VLM_REPAIR
DOWNGRADED_EXCLUDED
NEEDS_HUMAN_REVIEW
```

A row can become `RECOVERED_DEMO_CANDIDATE` only if:

- value is parseable or already valid;
- raw/normalized metric is usable;
- unit issue is repaired or correctly marked not applicable;
- period is present or repaired;
- source trace/evidence is present;
- no high-severity issue remains;
- no unresolved conflict exists;
- promotion remains demo-only and sidecar-only.

Rows with image evidence but unresolved header/value alignment should go to `NEEDS_VLM_REPAIR`, not recovered.

Rows with text/image/context conflicts should go to `NEEDS_HUMAN_REVIEW`.
