# 343L：Pure Human Attestation Result Ingestion

## 1. Positioning

343L ingests the filled pure-human attestation workbook generated by 343K.

The purpose is to validate and ingest the human attestation decisions for the 10 rows that were previously AI-assisted strict-review confirmed. If the filled workbook contains valid human accept/correct/reject/source-check/defer decisions, 343L must produce a scoped attestation result and audit gate.

343L is an attestation-result ingestion and audit-gating task. It is not production apply, not formal client export, not Argilla integration, not a frontend implementation, not real LLM/VLM review, and not an upstream extraction rerun.

Important scope boundary: even if all 10 rows are accepted by a human reviewer, that only completes pure-human confirmation for this 343K/343L package scope. It does not make the whole corpus production-ready or client-ready.

## 8. Core logic

### 8.1 Read filled attestation workbook

Read exactly one filled workbook from the input directory unless a `--filled-workbook` argument is supplied.

Validate:

- required sheet exists
- required identity columns exist
- editable human attestation columns exist
- identity is preserved versus 343K attestation items
- row count matches expected 10 unless explicitly allowed by input contract
- every filled row has an allowed decision

### 8.2 Validate human attestation decisions

Rules:

- `HUMAN_ACCEPT_AI_ASSISTED_CONFIRM` requires human reviewer id, reviewed date, `human_source_evidence_checked = true`, and `human_independent_check_attested = true`.
- `HUMAN_CORRECT` requires human reviewer id, reviewed date, corrected metric/year/value/unit, note, and `human_source_evidence_checked = true`.
- `HUMAN_REJECT` requires human reviewer id, reviewed date, note, and `human_source_evidence_checked = true`.
- `HUMAN_NEEDS_SOURCE_CHECK` requires human reviewer id, reviewed date, and note.
- `HUMAN_DEFER` requires human reviewer id and reviewed date; note is strongly recommended.

Invalid rows should make the milestone NOT_READY and produce validation errors.

### 8.3 Count decisions

Count:

- `human_accept_count`
- `human_correct_count`
- `human_reject_count`
- `human_needs_source_check_count`
- `human_defer_count`
- `valid_row_count`
- `invalid_row_count`
- `human_source_evidence_checked_true_count`
- `human_independent_check_attested_true_count`

For a fully accepted workbook, expected values are likely:

- human_accept_count = `10`
- human_correct_count = `0`
- human_reject_count = `0`
- human_needs_source_check_count = `0`
- human_defer_count = `0`
- valid_row_count = `10`
- invalid_row_count = `0`
- human_source_evidence_checked_true_count = `10`
- human_independent_check_attested_true_count = `10`

### 8.4 Package-level completion logic

If all 10 rows are valid and every row is either `HUMAN_ACCEPT_AI_ASSISTED_CONFIRM` or `HUMAN_CORRECT` with required evidence checks, set:

- `pure_human_attestation_result_ingested = true`
- `pure_strict_human_confirm_count = human_accept_count`
- `pure_strict_human_correct_count = human_correct_count`
- `pure_strict_human_reject_count = 0`
- `pure_strict_human_review_completed_for_package = true`
- `strict_human_review_completed_scope = 343K_PACKAGE_ONLY`
- `requires_pure_human_confirmation = false` for this package scope only
- `ready_for_343m = true`
- `recommended_343m_scope = human_confirmed_sidecar_apply_simulation_and_limited_export_gate`

If any row is rejected, source-check required, deferred, or invalid, set package completion false and recommend a remediation task.

### 8.5 Client export gate

Generate client export gate:

- `formal_client_export_allowed = false`
- `client_ready = false`
- `production_ready = false`
- `global_strict_human_review_completed = false`
- `package_strict_human_review_completed` based on package completion logic
- `strict_human_review_completed_scope = 343K_PACKAGE_ONLY` if completed
- `reason = Pure human attestation may be complete only for the 10-row 343K package; formal client export still requires a downstream scoped export gate and remaining backlog handling.`

### 8.6 Next action plan

If QA passes and package-level completion is true, recommend:

- `343M Human-confirmed Sidecar Apply Simulation And Limited Export Gate`

This task should simulate applying the 10 human-confirmed rows to a sidecar and generate a limited export gate without changing production data.

If there are validation errors or non-accepted rows, recommend re-filling/remediation before 343M.
