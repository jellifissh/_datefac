# Docs Directory

Purpose:
- Project documentation, task specs, runbooks, architecture notes, and handoff material.

Not responsible for:
- Importable runtime code.
- Large generated benchmark outputs.

Place new files here when:
- the change is documentation-first
- a numbered task needs a spec in `docs/codex_tasks/`
- a workflow needs a runbook in `docs/demo/`
- a repo or module boundary needs explanation in `docs/architecture/`

Category:
- documentation

MinerU-first / table-first relation:
- `docs/demo/` and `docs/codex_tasks/` record the MinerU benchmark chain, task boundaries, and safe operating guidance.
